# Code of Conduct

This project aims to make technical disagreement clear, rigorous, and respectful.

## Expected behavior

- critique code, claims, and artifacts without personal attacks
- keep examples and evidence precise
- assume good faith while staying honest about bugs and limitations
- help newcomers understand the workflow without condescension

## Unacceptable behavior

- harassment, intimidation, or discriminatory language
- abusive or demeaning conduct
- knowingly misrepresenting another person's work or evidence
- publishing private information without consent

## Reporting

Report conduct concerns to <mailto:jackgaff@umich.edu>. Reports will be handled discreetly.
